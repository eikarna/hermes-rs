## Responsibilities

- Review and merge pull requests
- Triage issues and bug reports
- Release new versions
- Maintain CI/CD pipelines
- Enforce the Code of Conduct

## Becoming a Maintainer

Contributors who consistently provide high-quality contributions over time may be invited to become maintainers. There is no formal process — maintainers are nominated by existing maintainers based on:

- Sustained contribution history
- Code quality and review thoroughness
- Helpful participation in issue discussions
- Alignment with project goals and conventions
